Local-DDx

Multi-Agent Collaborative Diagnostic System

A platform for testing and refining differential diagnosis experiments in ASI environments using local models. Local-DDx dynamically generates case-specific teams of AI specialist agents that collaborate through structured reasoning rounds — running entirely on local infrastructure to ensure complete patient data privacy.

View on GitHub How It Works

The 7-Round Diagnostic Pipeline

Modeled on how real multidisciplinary medical teams function: independent analysis, evidence-based discussion, adversarial challenge, and consensus-building.

1

Specialized Ranking

Individual agents assess their relevance to the clinical presentation and self-select into the diagnostic team.

2

Symptom Management

Independent recommendations for immediate triage interventions and symptom prioritization.

3

Team Differentials

Context-aware independent diagnosis generation using sliding context windows for peer awareness.

4

Master List

Synthesized compilation of all team diagnoses into a unified differential list with supporting evidence.

5

Refinement & Debate

Fully collaborative evidence-based discourse with Devil’s Advocate challenge mechanisms to stress-test diagnostic convergence.

6

Preferential Voting

Credibility-weighted Borda count consensus building for ranked diagnostic recommendations.

7

Can’t Miss

Critical diagnosis identification ensuring dangerous conditions are never overlooked, regardless of probability.

Key Features

Dynamic Specialist Generation

Autonomous creation of case-tailored agent teams without predefined specialty constraints. A dual-model architecture pairs conservative and innovative reasoning for balanced analysis.

Sliding Context Windows

Genuine collaborative reasoning where agents progressively see and respond to peer insights. Context filtering prioritizes information based on round type and agent specialty.

Consensus Mechanisms

Devil’s Advocate stress-testing of diagnostic convergence and preferential voting using credibility-weighted Borda counts for final recommendation synthesis.

Fully Local Processing

Complete patient data privacy through local-only infrastructure. No data leaves the machine. Deployable on CUDA GPUs or Apple Silicon via Ollama.

Comprehensive Evaluation

TempoScore metrics, credibility assessment, clinical recall and precision, diagnostic safety scoring, and round-by-round performance tracking.

Modular Architecture

Five composable modules — core, rounds, synthesis, evaluator, and runner — enabling targeted extension and rapid research iteration.
